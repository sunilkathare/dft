What is the VfM Database?

The VfM database is a central source of VfM data on investment proposals that are planned, in development or that have been approved.

This tool contains sensitive data. Circulation is restricted to those who are responsible for inputting or approving data into the database. The AAA team maintain a list of who has been given access to the database. If you wish to share the database with a colleague please notify AAA before doing so. Please note, the database should NOT be circulated outside the central Department. If this is considered necessary, please contact AAA to discuss.

What is the VfM information used for?

The VfM information is used to monitor, inform and support decision making across the department. It is regularly used for reporting on DfT's performance in VfM and has previously been used for analysis to shape the Department's Strategic Priorities (including the Transport Investment Strategy and Spending Review Bids).

This tool and the data collected in this update will be used to support preparations for the next Spending Review Bid.

How does it work?

The tool will allow you to review and update existing information and add information about new projects. This information is held on a central database managed by the AAA team. When an update is due, the latest version of the database will be sent out.

You can easily search for projects and create entries for new projects on the next page. Any individual can submit or edit information on the database. Any changes will be logged and attributed to that individual. You can find more information on how to use this excel tool in 'Instructions for database' (attached to the commissioning email).

Once all relevant information has been input or updated, a copy of the workbook should be sent to the AAA team so changes can be incorporated into the central database.

Please note, we treat the information provided as single source of truth. As such it should be signed off by a member of the SCS before being submitted. In 'Section 1: Project Management and Strategic Alignment' you are being asked whether the project has been signed off by the nominated SCS. Please ensure that you answer this question.

How long will it take to update the database?

You should allow roughly 20 minutes per project to input this into the database, once you have it to hand. Please note that you can also make changes in multiple sittings if you save this workbook locally, these can be submitted in the generated email once final.
